# Skill: 生成版本迭代说明

**说明**：
此 AI Skill 用于生成或更新项目根目录的 `功能迭代说明.csv`。当用户要求“生成版本迭代”“更新版本迭代说明”“写入功能迭代说明.csv”“新增迭代记录”等类似需求时，触发此技能。

该技能重点解决中文内容在 CSV 中显示乱码的问题：写入文件时必须使用 UTF-8 with BOM，也就是 Python 的 `utf-8-sig` 编码。这样双击用 Excel、WPS 或 Numbers 打开时，中文字段更容易被正确识别。

## 输出文件

默认输出路径：

```text
功能迭代说明.csv
```

默认写入位置为项目根目录。除非用户指定其他路径，不要写入 `pm_tools/` 或具体版本目录。

## 固定字段

CSV 表头固定为：

```csv
文件名称,提出时间,迭代内容,状态,修订人
```

字段含义：

| 字段 | 说明 | 示例 |
|------|------|------|
| 文件名称 | 对应版本目录或交付文件名称 | 智能外呼中台_demo_v1.0 |
| 提出时间 | 需求提出或版本记录日期 | 2026/5/18 |
| 迭代内容 | 本次版本的核心改动，建议用 `1.` `2.` 编号描述 | 1.新增数据概览模块；2.新增任务详情模块 |
| 状态 | 当前评审或交付状态 | 待原型评审 |
| 修订人 | 记录维护人 | 胡灏文 |

## 触发条件

当用户表达以下意图时，使用本技能：

- 生成版本迭代
- 更新版本迭代说明
- 新增版本迭代记录
- 写入 `功能迭代说明.csv`
- 生成 `功能迭代说明.csv`
- 维护版本计划
- 维护迭代计划

## 生成规则

1. 若用户提供完整记录，直接按固定字段生成或追加。
2. 若用户只提供版本名称和迭代内容，缺失字段可以参考现有项目习惯填写：
   - `提出时间`：优先使用用户给定日期；未给定时使用当天日期，格式为 `YYYY/M/D`。
   - `状态`：默认 `待原型评审`。
   - `修订人`：默认 `胡灏文`，除非用户另有说明。
3. `迭代内容` 中的多条内容建议合并在一个单元格内，使用中文分号 `；` 分隔。
4. 不要在 CSV 单元格内随意换行，避免表格软件打开后行数错乱。
5. 若更新已有文件，应尽量保留原有记录顺序，在末尾追加新记录；若用户要求重排，再按用户要求处理。
6. 若发现同名版本已存在，先更新同名记录，不要重复追加，除非用户明确要求保留多条历史记录。

## 编码要求

必须使用 UTF-8 with BOM 写入文件。

Python 写入示例：

```python
import csv

rows = [
    ["文件名称", "提出时间", "迭代内容", "状态", "修订人"],
    ["智能外呼中台_demo_v1.0", "2026/5/18", "1.新增数据概览模块；2.新增任务详情模块", "待原型评审", "胡灏文"],
]

with open("功能迭代说明.csv", "w", encoding="utf-8-sig", newline="") as f:
    writer = csv.writer(f)
    writer.writerows(rows)
```

注意：

- 不要使用 `encoding="utf-8"` 直接写 CSV，因为部分表格软件双击打开时仍可能乱码。
- 不要使用 GBK、GB2312 或系统默认编码。
- 不要手写拼接 CSV 行；应使用 CSV 写入工具，避免中文逗号、英文逗号、引号导致列错位。

## 读取旧文件

读取已有 `功能迭代说明.csv` 时，按以下顺序尝试：

1. `utf-8-sig`
2. `utf-8`
3. `gbk`

如果旧文件是制表符分隔内容，可以先识别为 TSV，读取后统一写回为标准 CSV。

读取时应兼容以下两种分隔符：

- 英文逗号 `,`
- 制表符 `\t`

写回时默认使用标准 CSV，即英文逗号分隔，并保留 UTF-8 BOM。

## 完成反馈

生成或更新完成后，应告知用户：

- 已写入的文件路径。
- 写入编码为 UTF-8 with BOM。
- 新增或更新了几条版本迭代记录。
