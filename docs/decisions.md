# 项目决策记录

## D-001 静态原型架构

使用 HTML、CSS 和原生 JavaScript，不引入构建工具，除非后续需求明确要求。

## D-002 业务资源分层

- 页面样式通过 `assets/css/app.css` 统一加载，具体规则按职责拆分在 `assets/css/`。
- Mock 数据统一维护在 `mock/data.js`。
- 公共工具、导航和启动入口分别维护在 `js/common.js`、`js/nav.js` 和 `js/app.js`。
- 用户洞察页面逻辑按职责维护在 `js/pages/userinsight/`。
- 功能标注统一维护在 `annotations/`。
- 产品文档和项目记忆分别维护在 `docs/` 与 `memory/`。

## D-003 兼容性原则

迁移只调整文件职责和引用路径，不改变既有 DOM、业务数据、全局函数名称和页面交互。

## D-004 培育工作台预测画像

- 预测画像作为电销员首次联系客户前的辅助信息，与通话后生成的 AI 画像区分。
- 业务要求的有效字段在右侧栏完整展示，不直接复刻原始表格。
- 字段按指标、键值、标签和信息卡呈现，并支持宽面板展开。
- 预测画像负责辅助判断，客户档案负责沉淀人工确认事实。
