3. 标签体系

  • 标签体系以智能体打标输出的客户标签集合为准，标签数据通过宽表中「客户标签集合」字段获取。
  • 智能体会根据通话内容自动为客户打标，涵盖客户基础信息、车辆与用车背景、购车意向与决策、购车关注点、抗拒点等多个维度。

4. 渠道编码

  • 渠道来源使用 R1 ~ R11 编码，代表 11 个不同投放来源渠道。
  • 意向等级使用 H（最高）、A、B、C、F、L、E、无效 八档分级。

5. 有效用户定义

  • 接通有效用户：原因类型为「接通有效」或「意向不明」的用户，排除打了无效标签的无效用户。
  • 无法接通用户：原因类型为「无法接通用户」的用户。

6. 客户关注点二级分类映射

智能体打标输出的原始标签，按以下规则归并为 4 个二级分类维度。每个客户对每个维度的判定逻辑为：该维度下至少有一个相关标签被标记为"是"或有值，即算该维度关注（有值）。

来源二级分类包含的原始标签
价格与优惠 关注店活动=是、关注补贴价格=是、关注贷款内容=是、关注车辆价格=是、关注店内优惠=是、关注金融优惠=是、关注互数服务=是、询问活动期限=是、询问活动覆盖范围=空、询问其他优惠=是、询问活动参与方式=是、询问参与活动车型=是、金蛋因子=是、询问车关注点包含"价格"
产品与配置 关注本品牌新车配置（智能化 / 舒适性 / 动力 / 辅助驾驶任一）= 是、关注新能源车型=是、购车关注点包含（安全性 / 保值性 / 品牌能耗 / 环保 / 保养）
服务与售后 关注评估内容=是、关注上牌政策=是、关注保养服务=是、关注售后服务=是、关注保险/参数=是、询问置换服务=是、购车关注点包含"服务"
竞品对比 关注竞品≠空
计算规则：
  1. 每个客户对每个维度只判断「是否关注」，依据为：该维度下至少有一个相关标签被标记为「是」（有值）。
  2. 样本中某维度的关注占比 = 关注该维度的客户数 / 有效客户总数。
  3. 各维度的关注度排名仅从高到低对比占比值，不受标签数量影响。
  4. 例如：价格与优惠关注占比 = 价格与关注的客户数 / 接通有效用户。

三、筛选栏说明

筛选栏位于页面顶部，单行放置 5 个筛选条件，右侧对齐「查询/重置」按钮。所有图表数据随筛选条件联动刷新。

查询逻辑：多个筛选条件之间为且（AND）关系，即同时满足所有已选条件的记录才会被纳入统计。下拉选项为「全部」时，表示不对该维度做过滤。

交互行为

  • 时间快捷按钮（#dateShortcuts）：点击「近 7 天」「近 30 天」「近 90 天」任一按钮，高亮选中态并自动更新内部日期区间。点击「自定义」展示开始/结束日期输入框（#customDateInputs）。
  • 线索等级多选下拉（#leadLevelMulti）：展开面板默认全选 8 项，可逐项勾选/取消。面板内包含「全选」快捷选项，支持一次全选/全清。面板外点击自动关闭。
  • 意向车系（#carSeries）：单选下拉，全部 / N7 / N6 / NX8 / 天籁鸿蒙。
  • 渠道名称（#rChannel）：单选下拉，全部 / R1 ~ R11。
  • 大项目输入框（#projectName）：支持键盘输入关键字模糊匹配，实时筛选候选项目名。
  • 查询按钮（蓝色 #queryBtn）：点击后触发全局数据联动刷新，所有看板图表根据当前筛选条件重新渲染。
  • 重置按钮（白底 #resetBtn）：将所有筛选项恢复为默认值（时间 → 近 30 天，其余 → 全部），并自动触发一次全量查询。

默认值

  • 时间范围：默认选中「近 30 天」快捷按钮
  • 线索等级：默认全选（8项，显示「全选 (8项)」）
  • 其余所有筛选项：默认「全部」（不限制该维度）

  • 查询（蓝色，#queryBtn）：点击后触发全局数据联动刷新，各看板图表根据当前筛选条件重新渲染（指标卡数值浮动 ±5% 模拟更新）。
  • 重置（白底，#resetBtn）：将时间范围恢复至「近 30 天」激活态，关闭自定义日期区域，所有下拉框归「全部」，文本输入框清空。

四、核心指标说明

位于筛选栏下方，以单行横向 Ribbon 视图展示 4 个分组、11 个核心 KPI，覆盖「触达 → 线索 → 用户分级 → 行为转化」全链路。容器类为 .kpi-cards-section > .kpi-group-row，4 个分组 flex 排列，各组宽度按比例伸缩。

布局结构：4 个 .kpi-group-compact 横向排列，每组有彩色左边框区分（蓝色=通话、紫色=线索、青色=用户分级、绿色=行为转化）。组内各指标以 .kpi-metric-item + 竖线分隔符 .kpi-metric-divider 并排展示，无需换行即可一屏展示全部 11 项指标。

五 · 1 渠道效果（channel-effect）

核心评估前端投放的进线质量、渠道分布及各地区获客成本。

核心构成与趋势
  • 意向车系、有效占比、线索级别： 采用环形图/饼图呈现大盘各分类比重，直观对比主力车系与潜客意向层级。
  • 区域投放效果： 采用横向双向条形图，左侧统计进线量，右侧统计建卡率，对比不同区域的转化效率。支持 Top 10 / Top 20 切换按钮。
  • 渠道线索质量： 采用多维度堆叠柱状图，直观展示每个子渠道在 H/A/B/C 等级的线索产出贡献。
  • 大项目 & 排期线索质量排名： 采用双列表布局并排展示，包含排名标签、项目名、总进线量及各意向级别的人数堆叠进度条。

区域投放效果 Top N 切换

  • 控制按钮位于区域投放图表右上角（#regionTopTabs），提供「Top 10」和「Top 20」两个切换选项。
  • 默认展示 Top 10；点击「Top 20」后，隐藏的 10 条额外数据行（.ce-v-bar-extra）展开显示。
  • 切换仅改变可见行数，不重新请求数据。

大项目 / 排期排名指标切换

  • 两个排名组件各有一组维度切换按钮（#rankMetricTabs / #scheduleMetricTabs），支持切换：HAB 占比 / 到店率 / 试驾率 / 锁单率。
  • 切换时 HAB 维度采用 H/A/B/Other 堆叠进度条展示，其余维度采用单色进度条。
  • 切换时列表先 opacity 归零 → 渲染新数据 → opacity 恢复，实现 300ms 过渡动画。
  • 每个列表展示 Top 10。

五 · 2 培育运营（cultivation-op）

页面采用单页垂直扁平化展开结构，取消了早期的阶段性子 Tab，全景式展示培育转化漏斗详情。

模块 1：用户基础属性
  • 卡片构成： 第一行 4 张卡片（年龄购车分布 / 收入水平构成 / 家庭构成分布 / 用车场景分布），第二行 2 张卡片（用车场景分布补充 / 品牌认知基础），采用 .kpi-row 横向弹性布局。
  • 年龄购车分布： 堆叠柱状图，X 轴为年龄段（18-25岁 / 25-35岁 / 35岁+），Y 轴为高度代表人数。每组柱子内分段显示男（蓝色 #3b82f6）/ 女（浅蓝 #93c5fd）/ 未知（灰色 #d1d5db）占比。悬停显示 Tooltip 展示具体数值和人数。
  • 收入水平构成： 环形饼图，中心显示用户总数，环形分段展示「富裕 25%」「一般 45%」「未提及 30%」三类，下方图例列出颜色标识和人数。
  • 家庭构成分布： 环形饼图，中心显示用户总数，环形分段展示「1至3人 55%」「4至5人 30%」「5人以上 15%」三类。
  • 用车场景分布： 横向进度条图，按用户数降序排列（城市通勤 85% / 家庭出行 72% / 自驾旅行 58% / 商务接待 42% / 亲子出行 35% / 物流运输 15%），每条左侧标签 + 中间进度条 + 右侧百分比和人数。
  • 品牌认知基础： 梯形漏斗图，从上到下递减展示「日产粉丝 15%」「基本了解 35%」「知道但不太了解 30%」「完全不了解 20%」，体现品牌认知深度递进关系。
  • 交互说明： 各卡片高度自适应，图表数据紧密关联用户自然属性，为后续沟通策略提供精准画像指引。年龄分布图悬停显示 Tooltip。
模块 2：购车意向分布 (Intent)
  • 卡片构成： 3 张卡片横向并排展示（购车意向分布 / 购买类型分布 / 计划购车分布），统一高度 160px。
  • 购车意向分布： 横向进度条，3 个分类：考虑中 45% / 明确 42% / 无意向 13%。考虑中和明确使用蓝色系，无意向使用浅蓝色区分。
  • 购买类型分布： 横向进度条，4 个分类：首购 52% / 换购 28% / 增购 15% / 未知 5%，按购买意愿从强到弱排列。
  • 计划购车分布： 横向进度条，5 个时间维度：一月内 35% / 三个月内 30% / 三月后 20% / 计划7天 10% / 未知 5%，按购车紧迫度从高到低排列。
模块 3：关注点热度 (Concern)
  • 整体关注排序： 横向条形图，4 个维度按提及率降序排列：产品与配置 78% / 价格与优惠 65% / 竞品对比 42% / 服务与售后 28%。点击任一条形触发 openFocusDrillDown(category)，打开关注点二级下钻弹窗。
  • 渠道关注点热度分布（Heatmap）： 5行×7列矩阵热力图。
    • Y 轴（行）： 4 个关注维度（产品与配置 / 价格与优惠 / 竞品对比 / 服务与售后）
    • X 轴（列）： 7 个渠道来源（R1 ~ R7）
    • 颜色编码： 深蓝色（#1e3a8a）= 高提及率，浅蓝/白色 = 低提及率
    • 悬停交互： 显示 Tooltip 展示「提及率: XX%」
    • 点击交互： 点击任一格子触发 openFocusDrillDown(category)，传入当前关注维度 category 参数
  • 热力图图例： 右上角显示渐变色条（低 → 高），标注「关注热度：低」和「高」文字说明。
模块 4：意向与阻力榜单 (Ranking)
  • 结构布局： 左右 1:1 并排展示，每张卡片右上角有「查看全部」按钮，点击后打开全量排行榜弹窗。
  • 质量标签排行榜： 展示原因类型（休眠失联/接通有效/再次联系/已接无效/空号停机等）、具体原因标签、用户数、占比、环比。默认展示 Top 10。
  • 抗拒点标签排行榜： 展示回访结果（战败/休眠未购/其他）、具体原因（如「价格因素」「资金不足」「半年内不购车」）、用户数、占比、环比。默认展示 Top 10。
  • 排名列样式： 前 3 名使用金银铜色徽章（.rank-1 金色 / .rank-2 银色 / .rank-3 铜色），其余灰色。
  • 原因类型/回访结果标签样式：
    • 休眠失联 = 蓝色（#eff6ff / #1e40af）
    • 接通有效 = 绿色（#f0fdf4 / #166534）
    • 再次联系 = 橙色（#fff7ed / #9a3412）
    • 已接无效 = 粉色（#fdf2f8 / #9d174d）
    • 空号/停机 = 灰色（#f3f4f6 / #374151）
    • 战败 = 红色（#fef2f2 / #dc2626）
    • 休眠未购 = 紫色（#f5f3ff / #7c3aed）
  • 具体原因列： 限制列宽，溢出文本自动省略号（.ellipsis）。
  • 环比列： 上升 ▲ + 红色 / 下降 ▼ + 绿色 / 持平 — + 灰色。
  • 「查看全部」按钮： 位于卡片头部右侧，文字按钮（.text-link-btn），点击触发 openFullRanking('quality') 或 openFullRanking('resistance')。

弹窗系统说明

关注点二级下钻弹窗（focusDrillDownModal）
  • 触发方式： 点击「整体关注排序」任意条形 或 「渠道关注点热力图」任意单元格，调用 openFocusDrillDown(category)，传入关注维度名称。
  • 弹窗结构：
    • 标题：[维度名] 二级分布 - 提及率排名（如「[产品与配置] 二级分布 - 提及率排名」）
    • 内容区：动态渲染该维度下的二级标签列表，每行显示标签名 + 进度条 + 提及率和人数
  • 数据来源： focusSubTagsData[category] 对象，存储各维度二级标签的提及率数据。
  • 关闭方式： 点击 × 按钮 或 点击弹窗遮罩层，调用 closeFocusModal()。
全量排行榜弹窗（rankingModal）
  • 触发方式： 点击质量标签排行榜或抗拒点标签排行榜卡片右上角的「查看全部」按钮，调用 openFullRanking('quality') 或 openFullRanking('resistance')。
  • 弹窗结构：
    • 标题：质量标签全量排行榜 或 抗拒点标签全量排行榜
    • 表头行（rankingModalThead）：排名 / 原因类型 / 具体原因标签 / 用户数 / 占比 / 环比
    • 表体（rankingModalTableBody）：动态渲染全量数据行
  • 数据来源： qualityFullData（15 条完整数据）和 resistanceFullData（12 条完整数据）。
  • 关闭方式： 点击 × 按钮 或 点击弹窗遮罩层，调用 closeRankingModal()。
洞察标签配置弹窗（insightModal）
  • 触发方式： 点击「洞察标签配置」按钮（#insightBtn），调用 initModal() 初始化弹窗内容。
  • 弹窗结构：
    • Tab 切换：「常见问题分类」/「客户抗拒点分类」
    • 左侧分类树：三级树形结构（1级展开/收起，2级展开/收起，3级叶子节点）
    • 右侧配置面板：显示当前选中节点的详情
  • 分类树节点操作：
    • 新增下级： 点击 + 按钮，展开表单填写节点名称和编码
    • 编辑： 点击编辑图标，修改节点名称和编码
    • 删除： 点击删除图标，二次确认后删除（含子节点一并删除）
  • 表单字段： 节点名称、节点编码、标签逻辑说明（三级节点专属）
  • 关闭方式： 点击 × 按钮 或 点击弹窗遮罩层，弹窗隐藏（不移除 DOM）
