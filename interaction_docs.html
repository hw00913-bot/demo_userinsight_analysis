版本 V3.0 · 与当前项目同步（2026-06-05 更新）

用户洞察 · 交互逻辑说明文档

基于当前 userinsight.html、script.js、mock.js 与 data/annotations.js 更新，覆盖渠道效果、培育运营、用户群体洞察三大分页，以及筛选、统计口径、下载、下钻弹窗和功能标注说明。

一、版本说明

二、页面结构

文件职责当前说明
index.html入口页项目入口，导航到主框架。
main.html主框架左侧菜单，包含「用户洞察」「交互说明」等入口。
userinsight.html核心业务页三大分页、筛选栏、KPI、图表、统计口径抽屉、弹窗容器、标注系统接入。
script.js页面交互逻辑分页切换、KPI 动态渲染、图表重绘、下钻弹窗、门店筛选、下载明细、标注动态锚点。
mock.js前端模拟数据模拟扎帳宽表（KPI/饼图/排行/渠道/媒体）和 e3s（门店/大区/小区）的前端演示数据。

加载顺序：userinsight.html 先加载 mock.js 和 script.js，页面底部再加载 data/annotations.js 与 annotations-tool/annotation-runtime.js。渠道效果 KPI 卡片由 renderKpiCards() 动态生成，标注锚点在动态 HTML 中输出。

三、数据与统计口径

真实系统数据来源

  • 报表 / 统计图 / KPI 指标数据：来自数据组维护的 扎帳宽表，包含线索编码、通话记录、线索等级分类、到店/试驾/锁单/交车等转化数据。
  • 专营店 / 门店基础数据：来自营销平台 e3s 基础数据，包含门店编码、门店名称、大区/小区/城市归属、省份归属等组织架构信息。

当前原型数据

  • 原型阶段由 mock.js 和 script.js 中的前端静态数据模拟上述真实数据源，用于演示和验证页面交互逻辑。
  • 标注数据统一由 data/annotations.js 维护，68 条标注覆盖三大分页，权限范围已统一为「总部管理员和电销主管都有权限查看」。
  • 统计口径抽屉中的说明直接写在 userinsight.html 底部，内容与扎帳宽表的真实字段定义对齐。

核心口径约定

口径项说明
去重规则线索类指标多按线索编码去重；用户群体洞察偏用户视角，按手机号去重。
有效号码线索级别不等于无效号码的外呼有效用户或线索。
接通通话时长大于 0 的通话记录。
H/A/B 线索线索等级属于 H-试驾排程单、H-试驾线索单、H-非试驾线索单、A、B 的用户或线索。
行为转化到店、试驾、锁单、交车按对应行为发生情况去重统计，占比分母多为新增线索或新增线索用户。
时间维度支持「转化维度」和「线索日期」切换；转化维度按行为发生时间统计，线索日期按线索创建日期统计。

四、三大分页与切换

页面顶部通过 .tab-btn 切换三个业务分页，对应的内容容器为 .tab-pane。

分页DOM id业务目标
渠道效果channel-effect查看前端渠道、媒体、项目、区域、触媒/触店相关的获客效果与线索质量。
培育运营cultivation-op查看总部培育线索、通话、跟进过程、购车意向、关注点和回访结果。
用户群体洞察user-group-insight查看新增线索用户和已接通用户的画像、品牌认知、收入、家庭、用户类型、用车场景和竞品关注。
切换逻辑：点击 Tab 后，脚本移除所有 .tab-pane.active，再为目标面板添加 active；筛选栏、KPI、图表内容均跟随面板展示。

五、筛选栏说明

三大分页都有独立筛选栏。当前为静态原型，查询/重置主要用于演示筛选交互和状态恢复。

分页筛选项特殊交互
渠道效果时间范围、意向车系、渠道名称、媒体名称、大项目、落地平台、筛选门店、线索等级、原因类型大类、原因类型、时间维度切换。支持下载明细；原因类型随原因类型大类联动；门店筛选打开门店弹窗。
培育运营时间范围、意向车系、渠道名称、媒体名称、大项目、落地平台、筛选门店、线索等级、原因类型大类、原因类型、时间维度切换、分类、清洗方式。分类为新线索/总部休眠/异地回流/休眠回流/暂败回流；清洗方式为人工和多个 AI 方式。
用户群体洞察时间范围、意向车系、渠道名称、媒体名称、大项目、落地平台、筛选门店、线索等级、原因类型大类、原因类型、时间维度切换。与渠道效果筛选结构保持一致，但当前没有下载明细按钮。

通用控件行为

  • 时间范围：默认近 30 天，支持近 7 天、近 30 天、近 90 天和自定义日期。
  • 多选下拉：落地平台、线索等级、分类、清洗方式使用自定义多选结构，支持全选和逐项勾选。
  • 门店筛选：点击「筛选门店」打开 storeFilterOverlay，支持按大小区或省市区选择。
  • 查询：收集当前分页筛选项后刷新展示数据，原型阶段主要保留按钮交互。
  • 重置：恢复当前分页默认筛选状态，不影响其它分页。
  • 下载明细：仅渠道效果页提供，点击 downloadChannelEffectDetail() 导出模拟明细表。

六、KPI 指标说明

三大分页均使用 .kpi-cards-section 展示横向 KPI 组。渠道效果 KPI 由脚本动态渲染，其它分页为静态 HTML。

分页指标组说明
渠道效果线索统计、通话统计、线索大类、用户分级、行为转化包含全渠道新增线索、总部培育新增线索、通话/接通/平均时长、有效/无效/无法建联、H/A/B 线索和到店/试驾/锁单/交车。
培育运营线索统计、通话统计、线索大类、用户分级、行为转化聚焦总部培育范围，新增分类和清洗方式筛选后用于对培育过程进行运营分析。
用户群体洞察线索统计、通话统计、线索大类、用户分级、行为转化从用户视角展示新增线索用户、有效/无效用户、H/A/B 用户和行为转化用户数。
当前文案：用户分级相关 KPI 已统一为「H/A/B 线索」和「H/A/B 线索占比」。

七、统计口径说明抽屉

  • 页面右上角「统计口径说明」按钮调用 openStatsModal()，从右侧打开 #stats-drawer。
  • 抽屉内通过 switchStatsTab(tabKey) 切换三个说明分页：渠道效果、培育运营、用户群体洞察。
  • 每个说明表仅保留「指标名称」和「指标定义」两列，去掉分析对象、卡片名称、维度组合、示例值等冗余字段。
  • 点击遮罩、关闭按钮或按 Esc 均可关闭抽屉。
口径页签包含内容
渠道效果用户概览 KPI、渠道效果分析、触媒/触店习惯与频率更新分析。
培育运营按线索维度 KPI、培育跟进过程分析、购车意向分布、关注点 TOP 排名、回访结果排名。
用户群体洞察用户概览 KPI、画像分布：年龄购车、品牌认知、收入水平、家庭构成、用户类型、用车场景、关注竞品。

八、渠道效果（channel-effect）

图表与分析模块
  • 三类饼图：意向车系占比、线索有效占比、线索级别占比。线索级别占比由 renderLeadLevelPie() 重绘图表内容。
  • 线索质量排名：大项目线索质量排名、媒体线索质量排名，支持按 H/A/B 占比、到店率、试驾率、锁单率切换。
  • 区域投放：城市投放效果、大区投放效果、小区投放效果均展示线索量和 H/A/B 质量结构。
  • 渠道线索质量：按 R1-R11 渠道展示线索等级构成。
  • 质量标签分布：按无效号码、无法建联、有效号码及其原因类型展示分布。
  • 触媒/触店习惯：支持渠道分析/媒体分析切换，覆盖首次/末次留资、留资重合度、下发/成交专营店、触达专营店数量、首触-成交专营店和留资频率。
下钻与弹窗
  • 城市投放可打开 cityStoreModal。
  • 渠道线索质量可打开 regionChannelModal。
  • 大项目/媒体排名可打开 projectDrillModal 或 scheduleDrillModal。
  • 查看全部调用 openFullRanking(type) 打开全量排行榜。
  • 首触-成交专营店查询可打开 firstTouchDealStoreModal。

九、培育运营（cultivation-op）

培育跟进过程
  • 顶部为总部培育与门店虚拟号跟进过程页签，调用 switchCultivationFollowProcess() 切换。
  • 总部培育包含回访线索、呼叫线索、接通线索、激活下发线索。
  • 门店虚拟号包含线索跟进、呼叫线索、接通线索等过程分析。
购车意向、关注点与回访结果
  • 购车意向分布包含购车意向、购买类型、计划购车三个卡片。
  • 关注点 Top 排名包含整体关注排序和渠道关注点热度分布，热力图单元格支持关注点下钻。
  • 回访结果分析展示战败、休眠未购等回访结论和具体原因，支持查看全部。

十、用户群体洞察（user-group-insight）

用户群体洞察复用用户概览 KPI，并补充面向画像和偏好的分布图表。

模块说明
年龄购车分布按年龄段展示用户数、占比，并按性别拆分。
品牌认知基础按品牌认知深度展示漏斗式分布。
收入水平构成按收入水平标签展示用户构成。
家庭构成分布按家庭结构标签展示用户构成。
用户类型按保客、潜客等用户类型展示分布。
用车场景按通勤、家庭出行等用车场景展示分布。
关注竞品展示用户关注竞品车型分布。

十一、弹窗系统

弹窗 / 抽屉触发方式说明
storeFilterOverlay筛选门店按钮按大小区或省市区选择门店，确认后回填按钮文案。
focusDrillDownModal关注点排序、热力图等展示关注点二级下钻内容。
cityStoreModal城市投放效果点击城市 → 小区 → 门店的意向等级分布。
regionChannelModal渠道线索质量点击大区 → 小区 → 门店三级 Tab，展示渠道质量分布。
projectDrillModal大项目排名点击大项目线索质量下钻，层级高于全量排行榜。
scheduleDrillModal媒体排名点击媒体质量下钻，结构与项目下钻一致。
rankingModal查看全部支持 quality、resistance、project、schedule、assignedStore、dealStore 等类型全量榜单。
firstTouchDealStoreModal首触-成交专营店查询结果展示首触专营店与成交专营店关系分析。
stats-drawer统计口径说明按钮右侧抽屉，包含渠道效果、培育运营、用户群体洞察三个口径分页。
