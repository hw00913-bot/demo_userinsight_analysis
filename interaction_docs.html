3. 标签体系

  • 标签体系以智能体打标输出的客户标签集合为准，标签数据通过宽表中「客户标签集合」字段获取。
  • 智能体会根据通话内容自动为客户打标，涵盖客户基础信息、车辆与用车背景、购车意向与决策、购车关注点、抗拒点等多个维度。

3.1 常见标签三级分类（购车行为洞察）

标签按三级分类树结构组织，支持按一级/二级/三级逐级筛选分析：

3.2 客户抗拒点三级分类（战败归因）

抗拒点按「决策主体-产品价值-购车环境」三维框架归类，每维按强/弱抗拒区分：

级别分类名称说明

三级 已购竞品 / 明确无购车计划 / 价格偏高 / 离家太远 / 无试驾车 等 具体抗拒行为标签

4. 渠道编码

  • 渠道来源使用 R1 ~ R11 编码，代表 11 个不同投放来源渠道。
  • 意向等级使用 H（最高）、A、B、C、F、L、E、无效 八档分级。

5. 有效用户定义

  • 接通有效用户：原因类型为「接通有效」或「意向不明」的用户，排除打了无效标签的无效用户。
  • 无法接通用户：原因类型为「无法接通用户」的用户。

6. 客户关注点二级分类映射

智能体打标输出的原始标签，按以下规则归并为 4 个二级分类维度。每个客户对每个维度的判定逻辑为：该维度下至少有一个相关标签被标记为"是"或有值，即算该维度关注（有值）。

来源二级分类包含的原始标签
价格与优惠 关注店活动=是、关注补贴价格=是、关注贷款内容=是、关注车辆价格=是、关注店内优惠=是、关注金融优惠=是、关注互数服务=是、询问活动期限=是、询问活动覆盖范围=空、询问其他优惠=是、询问活动参与方式=是、询问参与活动车型=是、金蛋因子=是、询问车关注点包含"价格"
产品与配置 关注本品牌新车配置（智能化 / 舒适性 / 动力 / 辅助驾驶任一）= 是、关注新能源车型=是、购车关注点包含（安全性 / 保值性 / 品牌能耗 / 环保 / 保养）
服务与售后 关注评估内容=是、关注上牌政策=是、关注保养服务=是、关注售后服务=是、关注保险/参数=是、询问置换服务=是、购车关注点包含"服务"
竞品对比 关注竞品≠空
计算规则：
  1. 每个客户对每个维度只判断「是否关注」，依据为：该维度下至少有一个相关标签被标记为「是」（有值）。
  2. 样本中某维度的关注占比 = 关注该维度的客户数 / 有效客户总数。
  3. 各维度的关注度排名仅从高到低对比占比值，不受标签数量影响。
  4. 例如：价格与优惠关注占比 = 价格与关注的客户数 / 接通有效用户。

三、筛选栏说明

筛选栏位于各主分页内部，渠道效果与培育运营维护各自独立的筛选条件，右侧对齐「查询/重置」按钮。当前为原型交互，查询/重置用于模拟数据刷新反馈。

查询逻辑：多个筛选条件之间为且（AND）关系，即同时满足所有已选条件的记录才会被纳入统计。下拉选项为「全部」时，表示不对该维度做过滤。

交互行为

  • 时间快捷按钮（.date-shortcuts）：点击「近 7 天」「近 30 天」「近 90 天」任一按钮，高亮选中态并自动更新本分页筛选栏内的日期区间。点击「自定义」展示开始/结束日期输入框。
  • 线索等级多选下拉（.custom-multi-select）：展开面板默认全选 10 项，可逐项勾选/取消。面板内包含「全选」快捷选项，支持一次全选/全清。面板外点击自动关闭。
  • 渠道效果筛选：时间范围、意向车系、线索等级、渠道名称、大区、小区、门店、大项目。
  • 培育运营筛选：时间范围、意向车系、线索等级、大区、小区、门店、标签来源。
  • 查询按钮（蓝色 .query-btn）：点击后触发当前分页筛选反馈。
  • 重置按钮（白底 .reset-btn）：仅重置当前分页内的筛选栏，不影响另一个分页。

默认值

  • 时间范围：默认选中「近 7 天」快捷按钮
  • 线索等级：默认全选（10项，显示「全选 (10项)」）
  • 其余所有筛选项：默认「全部」（不限制该维度）

五 · 1 渠道效果（channel-effect）

核心评估前端投放的进线质量、渠道分布及各地区获客成本。

核心构成与趋势
  • 意向车系、有效占比、线索级别： 采用环形图/饼图呈现大盘各分类比重，直观对比主力车系与潜客意向层级。

  • 媒体名称映射（全量排名用）：scheduleCodeMap 将媒体全名映射到 scheduleCode（6 项）
关键技术实现
  • 通用渲染函数：renderDrillContent(channels) — 媒体质量下钻复用此函数渲染大区→小区→门店结构
  • 通用门店卡片：generateStoreCard(store, areaLabel) — 项目和媒体质量下钻共享门店卡片 HTML 生成
  • Tab 切换：
    • 项目下钻：initProjectTabSwitch() 限定在 #projectDrillModalContent
    • 媒体质量下钻：initDrillTabSwitch('#scheduleDrillModalContent') 通用函数
  • 层级叠加：z-index 设为 1100（高于 rankingModal 的 1000），确保从全量排名打开时正确叠加显示
  • Hover 样式：排名行 hover 时显示蓝色浅底（#f0f5ff），提示可点击
关闭方式
  • 项目下钻：点击 × 按钮 或 点击遮罩层，调用 closeProjectDrillModal()
  • 媒体质量下钻：点击 × 按钮 或 点击遮罩层，调用 closeScheduleDrillModal()

六 · 5 全量排行榜弹窗（rankingModal）

全量排行榜支持 4 种类型，通过 openFullRanking(type) 统一入口打开，不同类型展示不同的表头和数据列。

四种类型对比
type 参数标题数据列数据来源支持行级下钻
'quality' 通话质量标签全量排行榜 排名 / 原因类型 / 具体原因标签 / 用户数 / 占比 / 环比 qualityFullData（15 条） 否
'resistance' 抗拒点标签全量排行榜 排名 / 回访结果 / 具体原因 / 用户数 / 占比 / 环比 resistanceFullData（12 条） 否
'project' 全量大项目线索质量排名 排名 / 项目名称 / HAB占比 / 到店率 / 试驾率 / 锁单率 动态生成 30 行 是 前 6 个项目名可点击下钻
'schedule' 全量媒体质量排名 排名 / 媒体名称 / HAB占比 / 到店率 / 试驾率 / 锁单率 动态生成 30 行 是 前 6 个媒体名可点击下钻
触发方式
  • 通话质量标签排行榜卡片右上角「查看全部」→ openFullRanking('quality')
  • 抗拒点标签排行榜卡片右上角「查看全部」→ openFullRanking('resistance')
  • 大项目线索质量排名组件「查看全部」→ openFullRanking('project')
  • 媒体质量排名组件「查看全部」→ openFullRanking('schedule')
下钻联动
  • 全量大项目排名中带 projectCode 的行可点击，触发 showProjectDrillModal(projectCode)，打开项目下钻弹窗（z-index 1100 叠加在 rankingModal 之上）
  • 全量媒体质量排名中带 scheduleCode 的行可点击，触发 showScheduleDrillModal(scheduleCode)，打开媒体质量下钻弹窗
  • 名称映射：通过 projectCodeMap（6 项）和 scheduleCodeMap（6 项）将全名映射到代码
页边距设置
  • 所有 4 种类型全量排行榜弹窗的 drawer-body 统一设置 padding: 0 60px（左右各 60px 空白）V2.4
关闭方式
  • 点击 × 按钮 或 点击遮罩层，调用 closeRankingModal()
