一、页面概览

  • 页面路径：user_insight v1.0/userinsight.html，从侧边栏「统计报表 → 用户洞察」进入。
  • 侧边栏「统计报表」分组默认展开，当前子菜单项「用户洞察」显示高亮激活状态（蓝色背景 + 右侧蓝色边框）。
  • 整体布局：左侧固定侧边栏（220px） + 右侧主内容区。主内容区从上到下依次为：顶部页头、筛选区、操作按钮栏、KPI 指标卡片、标签页内容。

二、筛选栏说明

1. 统计日期区间

  • 包含「开始日期」和「结束日期」两个日期输入框，中间以「至」连接。
  • 页面初始化时，日期区间自动设置为最近 30 天（结束日期为今天，开始日期为今天往前 30 天）。
  • 日期格式为 YYYY-MM-DD，使用原生日期选择器。

2. 呼叫类型

  • 下拉单选，可选项：全部、人工外呼、预外呼、AI 外呼。
  • 默认选中「全部」。

3. 业务类型

  • 下拉单选，可选项：全部、NEV新线索、NEV 冷线索、NEV 留资未满。
  • 默认选中「全部」。

4. R渠道

  • 下拉单选，可选项：全部、R1、R2、R3、R4。
  • 默认选中「全部」。
  • 用于按客户意向等级进行数据筛选。
  • 注意：当前选项为模拟数据，实际选项将根据业务需求配置。

5. 大项目名称

  • 文本输入框，支持关键字模糊匹配查询。
  • 输入提示文字：「输入关键字查询」。
  • 默认值为空（查询全部）。
  • 输入关键字后点击「查询」按钮进行筛选。

6. 查询与重置

  • 查询按钮：点击后触发全页数据刷新（原型中为静态展示，无实际接口调用）。
  • 重置按钮：点击后将所有筛选项恢复为默认值（日期恢复为最近 30 天，其他恢复「全部」）。

四、核心指标卡片

位于操作按钮栏下方，以 4 列网格（metrics-grid）排列，当屏幕较小时自动换行（minmax 280px）。

环比的定义

环比 = 与当前筛选日期区间等长的前一个时间窗口进行对比，非固定自然月。

例如：当筛选区间为 2026-03-03 至 2026-04-02（共 30 天），则对比基准期为 2026-02-01 至 2026-03-02（前 30 天）。

计算公式：环比变化率 = (当期值 - 基准期值) ÷ 基准期值 × 100%
当基准期值为 0 时，环比显示为「—」（持平/无法计算）。
指标名称图标颜色环比趋势说明
通话总数蓝色 (blue)↑ 12.5%统计周期内的全部通话量
接通总数绿色 (green)↑ 8.3%有效接通（通话时长 > 5s）的通话量
含常见问题通话数量橙色 (orange)↓ 5.2%通话内容命中至少一个常见问题标签的数量
含抗拒点通话数量红色 (red)— 0.0%通话内容命中至少一个抗拒点标签的数量
趋势图标规则：↑ 上升显示绿色（metric-trend up），↓ 下降显示红色（metric-trend down），— 持平显示灰色（metric-trend equal）。

hover 效果

  • 每张指标卡鼠标悬停时：向上位移 2px，阴影从 0 1px 3px 增强至 0 4px 12px。

五、标签页切换逻辑

  • 点击任意页签时：取消当前激活页签的 active 类，为点击的页签添加 active 类；对应的内容面板同步切换显示/隐藏。

六、常见标签页签详细说明

1. 报表统计区域

  • 位于页签顶部，以 2 列网格布局展示两个报表表格。
  • 按业务类型统计：展示不同三级标签在各业务类型（NEV新线索、NEV冷线索、NEV留资未满）下的通话数量和环比变化。
  • 按R渠道统计：展示不同三级标签在各R渠道（R1~R4）下的通话数量和环比变化。
  • 每张报表第一列为行标签（三级标签名称），单元格内显示绝对值 + 环比趋势图标。
  • 颜色规则：上升趋势（trend up）使用红色，下降趋势（trend down）使用绿色。
  • 报表支持纵向滚动（max-height: 280px）。

2. 标签分类分布（双环分布图）

  • 左侧列布局展示一个双环分布图，内环表示一级分类，外环表示二级分类。
  • 双环图数据从洞察标签树（common-issues-modal）动态读取，根据树结构中的一级/二级分类节点数量计算占比。
  • 由 JS 函数 initCommonTagCharts() 在弹窗打开时自动渲染。
  • 使用 SVG stroke-dasharray 技术实现，内环 r=11 stroke-width=5，外环 r=17 stroke-width=4。
  • 图例分为两组（「一级分类」/「二级分类」），带环层级指示点（内环实心点 / 外环空心点），显示名称、数量和占比百分比。

3. 常见标签排行表格

  • 位于右侧卡片，展示所有三级标签的排名数据。
  • 列字段：排名（角标）、常见标签名称、提及次数、占比（含进度条）、环比、热度。
  • 前三名排名角标（rank-badge）使用不同渐变色：第1名红色、第2名橙色、第3名黄色。
  • 占比列使用内嵌进度条可视化展示。
  • 热度值以橙色标签样式展示。

4. 排行排序切换功能

  • 卡片头部有排序切换按钮组（sort-switch-group），提供三种排序方式：
  • 按提及次数（默认激活）、按环比、按热度。
  • 由 JS 函数 initCommonTagSort() 处理排序逻辑。
  • 切换排序后，表格数据重新排序，前三名角标同步更新。
  • 问号图标悬停显示热度计算口径 tooltip：「热门问题是提及占比×100×60%+环比×100×40%得出热度得分」。

七、抗拒点页签详细说明

1. 报表统计区域

  • 位于页签顶部，以 2 列网格布局展示两个报表表格。
  • 按业务类型统计：展示不同抗拒点在各业务类型（NEV新线索、NEV冷线索、NEV留资未满）下的通话数量和环比变化。
  • 按R渠道统计：展示不同抗拒点在各R渠道（R1~R4）下的通话数量和环比变化。
  • 每张报表第一列为行标签（抗拒点名称），单元格内显示绝对值 + 环比趋势图标。
  • 颜色规则：上升趋势（trend up）使用红色，下降趋势（trend down）使用绿色。
  • 报表支持纵向滚动（max-height: 280px）。

2. 抗拒点排行

  • 以数据表格展示所有抗拒点的排名数据。
  • 列字段：排名（角标）、抗拒点名称、提及次数、占比（含进度条）、环比。
  • 前三名排名角标使用不同渐变色：第1名红色、第2名橙色、第3名黄色。
  • 占比列使用内嵌进度条可视化展示。

3. 排行排序切换功能

  • 卡片头部有排序切换按钮组（sort-switch-group），提供两种排序方式：
  • 按提及次数（默认激活）、按环比。
  • 由 JS 函数 initResistanceSort() 处理排序逻辑。
  • 切换排序后，表格数据重新排序，前三名角标同步更新。
注意：抗拒点报表中「环比」均指当前统计周期与上一个相同长度周期的对比，非固定月度环比。当前版本为静态模拟数据，生产环境需接入实际 API。

八、洞察标签管理弹窗

点击顶部操作栏「新增洞察标签」按钮（addInsightBtn）打开弹窗，ID 为 insightModal，宽度最大 1000px，最大高度 85vh。

1. 弹窗打开与关闭

  • 打开：点击「新增洞察标签」按钮，弹窗叠加层（modal-overlay）添加 active 类显示。
  • 关闭方式：① 点击右上角 ✕ 按钮；② 点击弹窗背景遮罩区域。

2. 主页签体系

  • 弹窗顶部有两个页签（modal-tab）：常见标签分类（common-issues-modal）和客户抗拒点分类（resistance-points-modal）。
  • 点击页签时：取消所有 modal-tab 的 active 类，激活点击的页签并显示对应内容面板（modal-content）。

3. 树形分类结构

分类层级底层节点类型
常见标签分类三级（一级分类 → 二级分类 → 三级标签）tree-level-3
客户抗拒点分类三级（一级分类 → 二级分类 → 三级标签）tree-level-3

4. 树节点展开/收起

  • 点击「树节点头部」（非操作按钮区域）可切换展开/收起状态：添加/移除节点的 open 类，子节点区域（tree-children）切换 display: block/none。
  • 展开时，左侧箭头图标（tree-icon）旋转 90°；收起时恢复 0°。
  • 使用全局事件委托处理，仅在 insightModal 处于 active 状态时生效。

5. 节点操作按钮（hover 显示）

  • 鼠标悬停于任意节点时，操作按钮（tree-actions）从透明渐变显示（opacity: 0 → 1）。
按钮图标作用显示条件
查看fa-eye打开「节点详情」弹窗，展示一/二/三级路径和标签逻辑说明底层节点（无子节点）
新增下级节点fa-plus打开「新增节点」表单弹窗非底层节点
编辑fa-pen打开「编辑节点」表单弹窗，预填当前数据所有节点
删除fa-trash（红色）弹出确认框，若有子节点则二次确认，确认后删除节点所有节点

6. 新增/编辑节点表单弹窗

  • 弹窗 ID：nodeFormModal，z-index 高于主弹窗（1100），保证层叠正确。
  • 表单字段：标签名（必填）、标签编码（必填，新增时自动生成随机编码并只读）、标签逻辑说明（底层节点时显示且必填）。
  • 自动编码生成逻辑：
    • 点击新增操作时，系统会自动基于 [A-Z, a-z, 0-9] 的大小写同字字符集随机抽取产生 6 位字母或数字组合。
    • 自动编码生成：新增节点时自动生成 6 位英文数字随机码（A-Z/a-z/0-9），常见标签和抗拒点标签统一适用，保证全局唯一。
  • 标题根据操作类型动态显示：「新增X级分类」或「编辑X级分类」。
  • 保存逻辑：新增时创建新 DOM 节点插入父级 tree-children 头部，编辑时更新对应 tree-label 文本内容和 data-logic-desc 属性。
  • 新增后自动展开：若父节点处于收起状态，新增子节点后父节点自动展开。

7. 节点详情弹窗

  • 弹窗 ID：nodeViewModal，展示字段：一级分类、二级分类、三级分类（客户抗拒点分类中隐藏）、标签逻辑说明。
  • 空值字段显示「-」并附 empty 样式。

8. 底部操作区（导入 / 导出 / 新增一级分类）

  • 「新增一级分类」：打开表单弹窗，新建的节点插入到树的顶部。
  • 「导入」（importCommonIssues / importResistancePoints）：弹出文件选择器，支持 .csv 文件。解析 CSV 后按编码匹配并更新现有节点的名称和逻辑说明，完成后弹出更新数量提示。
  • 「导出」（exportCommonIssues / exportResistancePoints）：将当前树结构序列化为 CSV 文件并下载。常见标签导出含七列（一/二/三级名称+编码+逻辑说明），抗拒点导出含五列（一/二级名称+编码+逻辑说明）。

9. 动态饼图联动

  • 当标签树数据发生变化（新增/编辑/删除节点）时，常见标签页签中的一级/二级分类饼图会自动重新渲染。
  • 由 initCommonTagCharts() 函数配合 MutationObserver 实现。
  • 弹窗关闭后重新打开时，饼图数据会更新为最新状态。

7. 节点详情弹窗

  • 弹窗 ID：nodeViewModal，展示字段：一级分类、二级分类、三级分类（客户抗拒点分类中隐藏）、标签逻辑说明。
  • 空值字段显示「-」并附 empty 样式。

8. 底部操作区（导入 / 导出 / 新增一级分类）

  • 「新增一级分类」：打开表单弹窗，新建的节点插入到树的顶部。
  • 「导入」（importCommonIssues / importResistancePoints）：弹出文件选择器，支持 .csv 文件。解析 CSV 后按编码匹配并更新现有节点的名称和逻辑说明，完成后弹出更新数量提示。
  • 「导出」（exportCommonIssues / exportResistancePoints）：将当前树结构序列化为 CSV 文件并下载。常见问题导出含七列（一/二/三级名称+编码+逻辑说明），抗拒点导出含五列（一/二级名称+编码+逻辑说明）。

十、人工复核流程完整说明

该功能模块负责对由「AI 智能体」初步打标的通话记录进行人工质量校验与标签修正，是确保用户洞察看板数据准确性的最终关卡。整个复核链路由两个独立页面构成。

业务闭环路径：AI 外呼标注 → 进入复核队列 → 人工抽检复核 → 生成流水存档 → 看板数据更新

2.4 命中情况 Tab（#hit-status）

  • 命中的常见问题：展示 AI 从通话中识别出的三级分类常见问题标签（如「全包落地含保险总价」、「置换旧车现金补贴细节」）。已命中项显示为 .review-tag.selected（蓝底蓝字蓝边框），未命中项为灰底灰字。
  • 命中的抗拒点：同样以标签形式展示，如「忙于工作私事」、「落地价高出预期」。
  • 标签点击交互：每个标签可点击切换「命中/不命中」状态，允许复核员直接在界面上实现精准纠错。
  • 复核意见：一个 height: 120px 的多行文本域，placeholder 提示「请输入复核意见...」，供复核员填写本次审核的判断依据。

2.5 历史复核记录 Tab（#history-records）

  • 标题栏右侧以「共 N 条」小字标注历史记录总数。
  • 每条记录内部结构依次为：复核人姓名 + 角色（如「张三 (Admin)」，粗体深色）、复核日期时间（如 2024-04-02 15:30，右对齐灰色）、复核流水 ID（如 LOG1000234，11px 极小字灰色，实际生产中可点击跳转至全局日志查询）、操作摘要 + 备注（如「操作：修正标签 / 备注：修正了抗拒点细分项」，多行 1.6 行距）。
  • 多条记录之间以 1px dashed #e2e8f0（虚线）分隔，视觉上形成时间轴效果。

2.6 底部提交栏（.submit-bar）

  • 固定在标注区底部，通过 margin-top: auto + Flex 布局实现自动下推定位，不受内容高度影响。
  • 「提交复核」按钮：btn-primary 蓝色实心，flex: 1 等宽，点击触发 submitReview() 函数。
  • 「下条通话」按钮：btn-secondary 白底边框，flex: 1 等宽，用于快速跳至队列中下一条记录，无需返回列表页，实现高效线性复核流程。

2.7 提交复核反馈机制

提交复核：点击「提交复核」按钮后，系统弹出成功提示并生成复核流水记录。生产环境此处应替换为 API 调用：将修正后的标签列表、复核意见、操作员信息推送至服务端，并触发看板数据重算。
数据优先级规则（生产规划）：人工复核提交后，该通话记录的标签数据优先级高于 AI 初始标注。看板「常见问题 Top 10」、「热门问题排行」、「抗拒点分布饼图」等所有模块的数据计算，应优先以经人工确认的标签为准。若一条通话被多次复核，以最新一次的流水记录为最终有效标注。
